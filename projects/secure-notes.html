Secure Login App

A secure Flutter-based authentication system built with Firebase to manage users, roles, and access permissions. Designed to strengthen my cybersecurity skills through practical, real-world development.

About This Project

This project started as a personal initiative to deepen my understanding of secure software design while preparing for the CompTIA Security+ certification. The app allows users to create accounts, sign in securely, and access content based on their roles using Firebase Authentication and Firestore.

Key Features

  • Firebase Authentication: Email/password login with form validation and error handling
  • Role-based access: Users are assigned roles (e.g., Admin, Standard User) to control access to specific pages
  • Secure data storage: User profiles and access roles are stored and fetched securely from Firestore
  • Scalable architecture: Modular folder structure and reusable components for long-term maintainability
  • Custom UI: Clean, user-friendly interface with theming options and state management

Technologies Used

Flutter Dart Firebase Auth Cloud Firestore State Management GitHub

Technical Implementation

The app uses Firebase Authentication for secure login and Cloud Firestore to store additional user information, including roles. Once authenticated, the user is redirected to a role-specific dashboard. Input fields include client-side and server-side validation to prevent unauthorized access or data issues.

Security Considerations

Security was central to this project. I implemented safe authentication flows, protected routes based on roles, and restricted Firestore access with Firebase security rules. This hands-on experience helped reinforce best practices while studying for the Security+ exam.

Future Enhancements

  • Multi-provider authentication (e.g., Google Sign-In)
  • Email verification and password reset functionality
  • Admin dashboard to manage users and roles
  • Encrypted local session storage
  • Deeper integration with analytics and monitoring
