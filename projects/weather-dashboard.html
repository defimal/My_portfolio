Construction Data Management System

A database-backed Java application developed as a student intern for the Nebraska Department of Transportation (NDOT) in 2025, focusing on transforming, validating, and transferring construction data between internal NDOT systems.

About This Project

This project involved building a robust full-stack Java application with an integrated relational database backend to manage large datasets related to construction metrics. The system streamlines data workflows within a company automating data transfer pipelines and ensuring accuracy and compatibility across multiple stages.

Key Features

  • Data transformation and validation: Ensured data integrity through multi-stage ETL processes
  • SQL data management: Wrote complex queries to manage and report on construction-related datasets
  • Python automation: Automated data transfer and processing pipelines for efficiency
  • C# integration: Interfaced with NDOT internal and legacy systems to ensure seamless data flow
  • Comprehensive documentation: Created Standard Operating Procedures (SOPs) for future interns and staff
  • Collaboration: Worked closely with NDOT IT team and data analysts to meet project requirements

Technologies Used

Java SQL Python C# JDBC ETL Processes

Technical Implementation

The application utilized Java for core development with JDBC for database connectivity, enabling efficient management of relational data. Python scripts automated ETL pipelines, improving reliability and reducing manual effort. C# components handled integration with NDOT‚Äôs legacy systems to ensure full system compatibility.

Project Impact

This internship project enhanced data accuracy, streamlined data workflows, and provided a reliable foundation for construction project reporting within NDOT. The work also involved producing detailed documentation to ensure knowledge transfer and ease of future maintenance.

Future Enhancements
